# HexSOC AI Engineering Memory

This folder is the engineering memory of HexSOC AI. It preserves architecture, product direction, engineering sessions, release notes, production incidents, and architecture decisions as the platform evolves from prototype into an enterprise cybersecurity technology company system.

HexSOC AI documentation should be treated as part of the product. Major engineering phases, meaningful architecture changes, and production learnings should be captured here before they disappear into chat history or local notes.

## Structure

- `docs/architecture/` stores system designs, platform diagrams, data flows, and security architecture.
- `docs/decisions/` stores Architecture Decision Records (ADRs) for important technical decisions.
- `docs/sessions/` stores phase-by-phase engineering logs from Codex and human development sessions.
- `docs/roadmap/` stores company, product, and platform roadmap planning.
- `docs/releases/` stores release notes and deployment summaries.
- `docs/incidents/` stores production debugging notes, outage records, and operational lessons learned.

## Documentation Standard

When a major phase is completed, add a session note under `docs/sessions/`.

When a meaningful architectural decision is made, add an ADR under `docs/decisions/`.

When production behavior changes, update architecture, release, or incident documentation as appropriate.
